# Access Rights

Access rights for the Role Catalogue is managed with a combination of the current User, a Section and a Permission.<br>
Each Section of the application demands a specific Permission, and only allows access to users with that permission.<br>
Permissions can also be constrained to a number of ItSystems and/or OrgUnits. In that case, the id of the entity needs
to be contained in the list of constrained id's on the specific permission for a user to have access.
